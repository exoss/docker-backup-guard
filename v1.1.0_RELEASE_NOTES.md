# Release Notes v1.1.0

**Date:** 2026-03-16

This release focuses on **maintenance and operational reliability**, with improvements to retention handling, security robustness, and CI publishing.

## 🚀 New Features

*   **Remote Retention Cleanup:** After a successful backup upload, the engine now removes remote archives older than `RETENTION_DAYS` and prunes empty directories.

## 🐛 Bug Fixes

*   **Retention Parsing Hardening:** Invalid or missing `RETENTION_DAYS` values now fall back to a safe default.
*   **Encryption Key Handling:** Improved error handling for unreadable key files and added coverage via unit tests.

## ⚙️ CI / Publishing

*   **Multi-Registry Publishing:** GitHub Actions workflow now supports pushing images to both GHCR and Docker Hub (when secrets are available).

## 🧹 Maintenance

*   **Code Cleanup:** Removed unused imports and reduced noisy warnings during HTTP operations.

## 📦 Installation / Update

To update to this version, pull the latest code/image or rebuild locally:

```bash
git pull
docker-compose down
docker-compose up -d --build
```
